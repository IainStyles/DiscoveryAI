\section{Exploring Clustering [10 marks]}

\begin{enumerate}
  \item A common use of clustering is to spatially ``segment'' image data, that is, to identify pixels that are spatially similar. In this exercise you will investigate the effect of different distance functions (similarity measures) and normalisation strategies on the performace of a cluatering algorithm. To begin, you will need to:

    \begin{enumerate}
      \item Load in the data file \texttt{unit4.mat}, which is the same dataset that was used in the dimensionality reduction exercise.
      \item Define anonymous functions that take two column vectors as input and compute the euclidean and cosine distances between the vectors respectively.
    \end{enumerate}

    For different combinations of distance measure, number of clusters and normalisation strategy:
    \begin{itemize}
      \item Perform $k$-means using the supplied code \texttt{kmeans.m}
      \item Plot the results using the supplied function\texttt{plotClusterImages.m}
    \end{itemize}

    For example, to run kmeans with 7 clusters and the cosine distance (defined as an anonymous function \texttt{dcos}), use:
   \begin{lstlisting}
     [cmean,ClusterIndex]=kmeans(X,7,dcos);
     plotClusterImages(ClusterIndex,nRows,nCols);
   \end{lstlisting}

    In your investigations I suggest you try:
    \begin{itemize}
      \item Different distances function (cosine, euclidean)
      \item Normalisation of samples (sum-of-squares, sum of components)
      \item Different numbers of clusters
      \item Using PCA to reduce the dimensionality before clustering (cluster on the PCA scores).
    \end{itemize}
    
    The challenge is: how well can you segment the data?

   \textit{\textbf{[Warning: $k$-means uses a random initialisation which means that it does not always give the same result. I suggest you run each clustering several times.]}}
\end{enumerate}
